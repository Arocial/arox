# 设计哲学与思考

本文档记录了 Arox 框架在设计初期关于 Agent 概念、LLM 能力边界以及与人类认知模型对比的一些思考。

## 核心概念

**Agent（智能体）**：具备某个特定领域或通用能力的实体，可以类比为人类在社会中扮演的某个角色。在 Arox 中，Agent 可以细分为以下几种类型：

- **基于规则的 Agent**：通过固定流程或工具实现特定需求。
- **基于 LLM 的 Agent**：结合大语言模型（LLM）的推理能力与外部工具来解决问题。
- **人类专家**：在某些环节中，人类本身也可以被视为一种特殊的 Agent 参与协作。

**Task（任务）**：需要完成的具体工作。任务会被分配给具备相应能力的 Agent 来执行。

## LLM Agent 的核心挑战

对于基于 LLM 的 Agent 来说，核心目标是如何让 LLM 最优地发挥其能力。这通常涉及以下几个层面的转化：

1. **Prompt 工程**：如何设计提示词以激发 LLM 的特定能力。
2. **信息上下文**：在具体实现中，这最终落地为：在发送给 LLM 的消息（Message）中，应该携带哪些上下文信息。
3. **Agent 关系建模**：是否能够通过对 Agent 之间的协作关系进行建模，使得某个 Agent 能够在特定时刻为 LLM 提供最优的上下文信息。

## LLM Agent 与人类认知的对比

如果我们将 LLM Agent 类比为人脑，可以将其建模为：**背景知识（训练好的模型权重） + 工作记忆（Prompt 上下文）**。

然而，LLM 与人类在认知机制上存在显著差异，这决定了我们在设计 Agent 架构时不能简单照搬人类的社会协作模式：

1. **知识的积累与更新**
    - **人类**：背景知识是逐步丰富的。在解决问题的过程中，与该问题相关的特定知识和经验会通过学习，逐步转化为长期的背景知识。
    - **LLM**：在单次请求中，模型的权重是固定的。我们只能将当前任务相关的信息整合到 Prompt（工作记忆）中。因此，系统设计的关键在于：**整合哪些信息，以及如何高效地整合这些信息**。

2. **多角色与能力边界**
    - **人类**：一个人可以具备多项能力并扮演多个角色（例如：工作中是工程师，业余是作家）。对于同一个人，不同的角色共享同一套背景知识（同一个大脑）。同时，不同的人拥有不同的背景知识和思维方式。
    - **LLM**：对于 LLM Bot 而言，似乎没有必要去模拟“同一个实体扮演不同角色”的模式。在架构设计上，让单个 Bot 专注于某一种特定能力（单一职责）往往更有效。核心依然回到了上一条：如何为这个专注的 Bot 提供最合适的上下文信息。

3. **记忆与表征机制**
    - **人类**：大脑的长时记忆与短时记忆并不是完全割裂的，它们在认知过程中紧密交织。
    - **LLM**：虽然 LLM 的模型权重（长时记忆）和 Prompt（短时记忆）也不是完全割裂的，但两者的融合方式与人脑有着本质的不同。我们需要通过外部系统（如 RAG、记忆管理模块）来弥补这种差异。
